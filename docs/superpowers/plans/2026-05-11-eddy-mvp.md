# eddy MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the eddy MVP — a mobile-portrait videosong creator where the layout-builder grid is the song. Each cell holds one recorded video+audio "voice"; pressing play composites all voices into one synced video+audio mix. Export deferred to v2.

**Architecture:** Single Vite app, flat layout. WebCodecs decode (`VideoDecoder` / `AudioDecoder`) + mediabunny demux + Web Audio scheduling + WebGL composite. Code is ported and pruned from `../eddy-1.0/` where battle-tested implementations exist; orchestration (transport, anchor-take, multi-cell coordination) is written fresh because the eddy-1.0 versions are over-engineered for this MVP.

**Tech Stack:** Solid 2.x (`solid-js`, `@solidjs/signals`, `@solidjs/web`), Vite, TypeScript, `@bigmistqke/view.gl` (WebGL), `mediabunny` (mux/demux), WebCodecs, Web Audio, `getUserMedia`, `MediaRecorder`, Playwright (browser-based tests).

**Reference:** Design spec at `docs/superpowers/specs/2026-05-11-eddy-mvp-design.md`. Source-of-truth for scope; resolve any conflict between this plan and the spec in favour of the spec.

**Three phases** — each phase ends in working software and a green test run.

- **Phase 0 — Setup.** Dependencies, test fixtures, Playwright fake-media config, Entity ID.
- **Phase 1 (M1) — Single-cell record + playback.** Ports + minimal pipeline + toolbar UI. Tap record into selected cell, tap stop, tap play, recording plays back.
- **Phase 2 (M2) — Multi-cell synced playback.** WebGL grid compositor over layout; multiple voices play in sync from the audio master clock.
- **Phase 3 (M3) — Anchor-take semantics.** First clip sets song length; subsequent recordings clamped; existing voices play during record; re-record/delete semantics.

```typescript
import { audioContext } from "./audio-context"

/**
 * Decode all audio chunks from a demuxed blob into a single AudioBuffer.
 * Suitable for short clips (seconds to minutes) where holding the whole
 * thing decoded in memory is fine.
 */
export async function decodeToAudioBuffer(
  chunks: EncodedAudioChunk[],
  config: AudioDecoderConfig,
): Promise<AudioBuffer> {
  const ctx = audioContext()
  const decoded: AudioData[] = []

  const decoder = new AudioDecoder({
    output: (data) => decoded.push(data),
    error: (e) => { throw e },
  })
  decoder.configure(config)

  for (const chunk of chunks) {
    decoder.decode(chunk)
  }
  await decoder.flush()
  decoder.close()

  if (decoded.length === 0) {
    throw new Error("decodeToAudioBuffer: decoder produced no frames")
  }

  const sampleRate = decoded[0].sampleRate
  const numberOfChannels = decoded[0].numberOfChannels
  const totalFrames = decoded.reduce((sum, d) => sum + d.numberOfFrames, 0)

  const buffer = ctx.createBuffer(numberOfChannels, totalFrames, sampleRate)

  let offset = 0
  for (const data of decoded) {
    for (let ch = 0; ch < numberOfChannels; ch++) {
      const dst = buffer.getChannelData(ch).subarray(offset)
      data.copyTo(dst, { planeIndex: ch, format: "f32-planar" })
    }
    offset += data.numberOfFrames
    data.close()
  }

  return buffer
}
```

```typescript
export function frameAtTime(decoder: VideoDecoderState, tMicros: number): VideoFrame | null {
```

```typescript
import { decodeToAudioBuffer } from "../media/audio-decoder"
import { demuxBlob, type DemuxResult } from "../media/demuxer"
import { makeVideoDecoder, type VideoDecoderState } from "../media/video-decoder"

export type Clip = {
  /** Cell id this clip belongs to. */
  cellId: string
  /** Duration in seconds. */
  duration: number
  /** Decoded audio. */
  audio: AudioBuffer
  /** Encoded video chunks + decoder state for on-demand playback. */
  video: VideoDecoderState
}

/**
 * Build a Clip from a recorded Blob. Demuxes, decodes audio to a single
 * AudioBuffer, prepares a VideoDecoder seeded with the encoded chunks.
 */
export async function blobToClip(cellId: string, blob: Blob): Promise<Clip> {
  const demuxed: DemuxResult = await demuxBlob(blob)
  const audio = await decodeToAudioBuffer(demuxed.audio.chunks, demuxed.audio.config)
  const video = makeVideoDecoder(demuxed.video.chunks, demuxed.video.config)
  return {
    cellId,
    duration: Math.max(audio.duration, demuxed.video.durationSeconds),
    audio,
    video,
  }
}

export function disposeClip(clip: Clip): void {
  clip.video.close()
}
```

- [ ] **Step 2: Add a `demuxBlob` wrapper at the bottom of demuxer.ts**

The ported `src/media/demuxer.ts` exposes the eddy-1.0 demuxer's API, which may be more complex than we need. Add a small, MVP-shaped wrapper at the bottom of `src/media/demuxer.ts`:

```typescript
import {
  Input,
  BlobSource,
  type AudioCodec,
  type VideoCodec,
} from "mediabunny"

export type DemuxResult = {
  audio: { chunks: EncodedAudioChunk[]; config: AudioDecoderConfig }
  video: { chunks: EncodedVideoChunk[]; config: VideoDecoderConfig; durationSeconds: number }
}

export async function demuxBlob(blob: Blob): Promise<DemuxResult> {
  const input = new Input({ source: new BlobSource(blob), formats: ["matroska", "webm", "mp4"] as any })

  const audioTrack = await input.getPrimaryAudioTrack()
  const videoTrack = await input.getPrimaryVideoTrack()
  if (!audioTrack) throw new Error("demuxBlob: no audio track")
  if (!videoTrack) throw new Error("demuxBlob: no video track")

  const audioChunks: EncodedAudioChunk[] = []
  for await (const sample of audioTrack.packets()) {
    audioChunks.push(new EncodedAudioChunk({
      type: sample.type === "key" ? "key" : "delta",
      timestamp: sample.timestamp * 1_000_000,
      duration: (sample.duration ?? 0) * 1_000_000,
      data: sample.data,
    }))
  }

  const videoChunks: EncodedVideoChunk[] = []
  for await (const sample of videoTrack.packets()) {
    videoChunks.push(new EncodedVideoChunk({
      type: sample.type === "key" ? "key" : "delta",
      timestamp: sample.timestamp * 1_000_000,
      duration: (sample.duration ?? 0) * 1_000_000,
      data: sample.data,
    }))
  }

  return {
    audio: {
      chunks: audioChunks,
      config: {
        codec: audioTrack.codec,
        sampleRate: audioTrack.sampleRate,
        numberOfChannels: audioTrack.numberOfChannels,
      },
    },
    video: {
      chunks: videoChunks,
      config: {
        codec: videoTrack.codec,
        codedWidth: videoTrack.codedWidth,
        codedHeight: videoTrack.codedHeight,
      },
      durationSeconds: await videoTrack.computeDuration(),
    },
  }
}
```

**Note:** mediabunny's exact API may differ slightly from the above — `Input`, `BlobSource`, the `packets()` iterator, and track-property names should match what mediabunny 1.27 exposes. If TypeScript complains, read `node_modules/mediabunny/dist/*.d.ts` to find the correct surface; the *shape* of `DemuxResult` is what consumers depend on, so keep that and adjust the inner implementation.

```typescript
import { createSignal } from "solid-js"
import { disposeClip, type Clip } from "./clip"

export type ClipStore = {
  clips: () => Record<string, Clip>
  setClip: (cellId: string, clip: Clip) => void
  clearClip: (cellId: string) => void
  getClip: (cellId: string) => Clip | undefined
  clear: () => void
}

export function createClipStore(): ClipStore {
  const [clips, setClips] = createSignal<Record<string, Clip>>({}, { ownedWrite: true })

  return {
    clips,
    setClip(cellId, clip) {
      setClips((prev) => {
        const existing = prev[cellId]
        if (existing) disposeClip(existing)
        return { ...prev, [cellId]: clip }
      })
    },
    clearClip(cellId) {
      setClips((prev) => {
        const existing = prev[cellId]
        if (existing) disposeClip(existing)
        const { [cellId]: _, ...rest } = prev
        return rest
      })
    },
    getClip(cellId) {
      return clips()[cellId]
    },
    clear() {
      const all = clips()
      for (const id in all) disposeClip(all[id])
      setClips({})
    },
  }
}
```

### Task 1.12: Write single-clip transport

**Files:**
- Create: `src/song/transport.ts`

For Phase 1 the transport is dumb: play a single clip from t=0 against the audio clock. Phase 2 will generalize it to N clips.

- [ ] **Step 1: Write transport.ts**

Create `src/song/transport.ts`:

```typescript
import { createSignal } from "solid-js"
import { audioContext } from "../media/audio-context"
import type { Clip } from "./clip"

export type TransportState = "stopped" | "playing"

export type Transport = {
  state: () => TransportState
  /** Audio-clock time in seconds since the current play started.
   *  0 when stopped. */
  position: () => number
  play: (clips: Clip[]) => void
  stop: () => void
}

export function createTransport(): Transport {
  const [state, setState] = createSignal<TransportState>("stopped", { ownedWrite: true })
  const [startedAt, setStartedAt] = createSignal(0, { ownedWrite: true })
  const sources: AudioBufferSourceNode[] = []

  function play(clips: Clip[]) {
    if (state() === "playing") stop()
    if (clips.length === 0) return

    const ctx = audioContext()
    const when = ctx.currentTime + 0.05
    for (const clip of clips) {
      const src = ctx.createBufferSource()
      src.buffer = clip.audio
      src.connect(ctx.destination)
      src.start(when)
      sources.push(src)
    }
    setStartedAt(when)
    setState("playing")

    const longest = Math.max(...clips.map((c) => c.duration))
    // Auto-stop at end (no loop for M1; M2/M3 will revisit).
    setTimeout(() => stop(), (longest + 0.1) * 1000)
  }

  function stop() {
    for (const src of sources) {
      try { src.stop() } catch {}
    }
    sources.length = 0
    setState("stopped")
    setStartedAt(0)
  }

  function position() {
    if (state() !== "playing") return 0
    return audioContext().currentTime - startedAt()
  }

  return { state, position, play, stop }
}
```

### Task 1.13: Single-cell compositor

**Files:**
- Create: `src/song/compositor.ts`

WebGL canvas drawing the current frame of one clip. Phase 2 will generalize to N cells.

- [ ] **Step 1: Write compositor.ts**

Create `src/song/compositor.ts`:

```typescript
import type { Clip } from "./clip"
import { frameAtTime } from "../media/video-decoder"

export type Compositor = {
  canvas: HTMLCanvasElement
  /** Draw the frame at time t (seconds) for each clip into the canvas
   *  at the given rect (canvas-pixel coordinates). */
  draw: (entries: Array<{ clip: Clip; rect: { x: number; y: number; w: number; h: number } }>, t: number) => void
  resize: (w: number, h: number) => void
  dispose: () => void
}

export function createCompositor(): Compositor {
  const canvas = document.createElement("canvas")
  const gl = canvas.getContext("webgl2", { alpha: false, premultipliedAlpha: false })
  if (!gl) throw new Error("WebGL2 unavailable")

  // Minimal shader + program. For M1 the texture draw is single-cell;
  // M2 keeps the same code but loops over multiple entries. Each entry's
  // rect becomes a quad in NDC.
  const vert = `#version 300 es
in vec2 a_pos;
in vec2 a_uv;
out vec2 v_uv;
void main() {
  v_uv = a_uv;
  gl_Position = vec4(a_pos, 0.0, 1.0);
}`
  const frag = `#version 300 es
precision mediump float;
in vec2 v_uv;
uniform sampler2D u_tex;
out vec4 outColor;
void main() {
  outColor = texture(u_tex, v_uv);
}`

  function compile(type: number, src: string): WebGLShader {
    const sh = gl!.createShader(type)!
    gl!.shaderSource(sh, src)
    gl!.compileShader(sh)
    if (!gl!.getShaderParameter(sh, gl!.COMPILE_STATUS)) {
      throw new Error(gl!.getShaderInfoLog(sh) ?? "shader compile failed")
    }
    return sh
  }

  const prog = gl.createProgram()!
  gl.attachShader(prog, compile(gl.VERTEX_SHADER, vert))
  gl.attachShader(prog, compile(gl.FRAGMENT_SHADER, frag))
  gl.linkProgram(prog)
  gl.useProgram(prog)

  const posLoc = gl.getAttribLocation(prog, "a_pos")
  const uvLoc = gl.getAttribLocation(prog, "a_uv")
  const tex = gl.createTexture()!
  gl.bindTexture(gl.TEXTURE_2D, tex)
  gl.texParameteri(gl.TEXTURE_2D, gl.TEXTURE_WRAP_S, gl.CLAMP_TO_EDGE)
  gl.texParameteri(gl.TEXTURE_2D, gl.TEXTURE_WRAP_T, gl.CLAMP_TO_EDGE)
  gl.texParameteri(gl.TEXTURE_2D, gl.TEXTURE_MIN_FILTER, gl.LINEAR)
  gl.texParameteri(gl.TEXTURE_2D, gl.TEXTURE_MAG_FILTER, gl.LINEAR)

  const buffer = gl.createBuffer()!
  gl.bindBuffer(gl.ARRAY_BUFFER, buffer)

  function draw(entries: Array<{ clip: Clip; rect: { x: number; y: number; w: number; h: number } }>, t: number) {
    gl!.clearColor(0, 0, 0, 1)
    gl!.clear(gl!.COLOR_BUFFER_BIT)

    const cw = canvas.width
    const ch = canvas.height
    for (const { clip, rect } of entries) {
      const frame = frameAtTime(clip.video, t * 1_000_000)
      if (!frame) continue

      gl!.bindTexture(gl!.TEXTURE_2D, tex)
      gl!.texImage2D(gl!.TEXTURE_2D, 0, gl!.RGBA, gl!.RGBA, gl!.UNSIGNED_BYTE, frame as any)

      // Convert rect (pixel coords) to NDC quad.
      const x0 = (rect.x / cw) * 2 - 1
      const y0 = 1 - (rect.y / ch) * 2
      const x1 = ((rect.x + rect.w) / cw) * 2 - 1
      const y1 = 1 - ((rect.y + rect.h) / ch) * 2

      const verts = new Float32Array([
        x0, y0, 0, 0,
        x1, y0, 1, 0,
        x0, y1, 0, 1,
        x1, y1, 1, 1,
      ])
      gl!.bindBuffer(gl!.ARRAY_BUFFER, buffer)
      gl!.bufferData(gl!.ARRAY_BUFFER, verts, gl!.STREAM_DRAW)
      gl!.enableVertexAttribArray(posLoc)
      gl!.enableVertexAttribArray(uvLoc)
      gl!.vertexAttribPointer(posLoc, 2, gl!.FLOAT, false, 16, 0)
      gl!.vertexAttribPointer(uvLoc, 2, gl!.FLOAT, false, 16, 8)
      gl!.drawArrays(gl!.TRIANGLE_STRIP, 0, 4)
    }
  }

  function resize(w: number, h: number) {
    canvas.width = w
    canvas.height = h
    gl!.viewport(0, 0, w, h)
  }

  function dispose() {
    gl!.deleteProgram(prog)
    gl!.deleteBuffer(buffer)
    gl!.deleteTexture(tex)
  }

  return { canvas, draw, resize, dispose }
}
```

Note on `texImage2D` accepting `VideoFrame`: Chromium supports passing `VideoFrame` directly as the pixel source. The `frame as any` cast is acceptable here (TS lib types don't cover it cleanly).

```tsx
import { createSignal, onCleanup, Show, useContext } from "solid-js"
import { Context } from "./context"
import { startCapture, type CaptureHandle } from "./media/capture"
import { blobToClip } from "./song/clip"
import { createCompositor } from "./song/compositor"
import styles from "./song.module.css"

export function Song() {
  const ctx = useContext(Context)
  if (!ctx) throw new Error("Song must be used inside Context")

  const [capture, setCapture] = createSignal<CaptureHandle | null>(null, { ownedWrite: true })

  function selectedCellId(): string | null {
    const sel = ctx.app.selection
    if (!sel) {
      const root = ctx.app.layout
      return root.type === "entity" ? root.id : null
    }
    // Resolve selection path to entity id.
    let node: any = ctx.app.layout
    for (const i of sel.path) {
      node = node.children?.[i]
      if (!node) return null
    }
    return node.type === "entity" ? node.id : null
  }

  async function onRecord() {
    const cellId = selectedCellId()
    if (!cellId) return
    const handle = await startCapture()
    setCapture(handle)
  }

  async function onStop() {
    const handle = capture()
    if (!handle) return
    const cellId = selectedCellId()
    setCapture(null)
    const blob = await handle.stop()
    if (cellId) {
      const clip = await blobToClip(cellId, blob)
      ctx.clips.setClip(cellId, clip)
    }
  }

  function onPlay() {
    const cellId = selectedCellId()
    if (!cellId) return
    const clip = ctx.clips.getClip(cellId)
    if (!clip) return
    ctx.transport.play([clip])
    runRenderLoop([clip])
  }

  function onStopTransport() {
    ctx.transport.stop()
  }

  // --- Compositor canvas + render loop (single cell for M1) ---
  const compositor = createCompositor()
  let compositorRef: HTMLDivElement | undefined
  onCleanup(() => compositor.dispose())

  function fitCanvas() {
    if (!compositorRef) return
    compositor.resize(compositorRef.clientWidth, compositorRef.clientHeight)
  }
  window.addEventListener("resize", fitCanvas)
  onCleanup(() => window.removeEventListener("resize", fitCanvas))

  let raf = 0
  function runRenderLoop(clips: any[]) {
    cancelAnimationFrame(raf)
    function tick() {
      if (ctx!.transport.state() !== "playing") {
        compositor.draw([], 0)
        return
      }
      const t = ctx!.transport.position()
      // M1: draw clip filling whole canvas.
      compositor.draw(
        clips.map((c) => ({
          clip: c,
          rect: { x: 0, y: 0, w: compositor.canvas.width, h: compositor.canvas.height },
        })),
        t,
      )
      raf = requestAnimationFrame(tick)
    }
    tick()
  }
  onCleanup(() => cancelAnimationFrame(raf))

  return (
    <>
      <div
        ref={(el) => {
          compositorRef = el
          el.appendChild(compositor.canvas)
          queueMicrotask(fitCanvas)
        }}
        class={styles.compositor}
      />
      <div class={styles.toolbar}>
        <Show
          when={capture() !== null}
          fallback={
            <button class={styles.button} onClick={onRecord}>
              ● Record
            </button>
          }
        >
          <button class={`${styles.button} ${styles.recording}`} onClick={onStop}>
            ■ Stop recording
          </button>
        </Show>
        <Show when={ctx.transport.state() === "stopped"} fallback={
          <button class={styles.button} onClick={onStopTransport}>■ Stop</button>
        }>
          <button class={styles.button} onClick={onPlay}>▶ Play</button>
        </Show>
      </div>
    </>
  )
}
```

### Task 3.3: Looping playback at songLength

**Files:**
- Modify: `src/song/transport.ts`

The Phase 1 transport stops after the longest clip. In M3, playback should *loop* at `songLength`.

- [ ] **Step 1: Add loop signal and update play**

Replace `src/song/transport.ts`:

```typescript
import { createSignal } from "solid-js"
import { audioContext } from "../media/audio-context"
import type { Clip } from "./clip"

export type TransportState = "stopped" | "playing"

export type Transport = {
  state: () => TransportState
  position: () => number
  /** Plays clips and loops at `loopLength` (seconds). If null, plays
   *  once and stops. */
  play: (clips: Clip[], loopLength: number | null) => void
  stop: () => void
}

export function createTransport(): Transport {
  const [state, setState] = createSignal<TransportState>("stopped", { ownedWrite: true })
  const [startedAt, setStartedAt] = createSignal(0, { ownedWrite: true })
  let sources: AudioBufferSourceNode[] = []
  let loopTimer = 0

  function schedule(clips: Clip[], when: number) {
    const ctx = audioContext()
    sources = []
    for (const clip of clips) {
      const src = ctx.createBufferSource()
      src.buffer = clip.audio
      src.connect(ctx.destination)
      src.start(when)
      sources.push(src)
    }
  }

  function play(clips: Clip[], loopLength: number | null) {
    if (state() === "playing") stop()
    if (clips.length === 0) return

    const ctx = audioContext()
    const when = ctx.currentTime + 0.05
    schedule(clips, when)
    setStartedAt(when)
    setState("playing")

    if (loopLength !== null) {
      const cycle = () => {
        if (state() !== "playing") return
        const ctx2 = audioContext()
        const next = ctx2.currentTime + 0.01
        for (const src of sources) {
          try { src.stop() } catch {}
        }
        schedule(clips, next)
        setStartedAt(next)
        loopTimer = window.setTimeout(cycle, loopLength * 1000)
      }
      loopTimer = window.setTimeout(cycle, loopLength * 1000)
    } else {
      const longest = Math.max(...clips.map((c) => c.duration))
      loopTimer = window.setTimeout(() => stop(), (longest + 0.1) * 1000)
    }
  }

  function stop() {
    for (const src of sources) {
      try { src.stop() } catch {}
    }
    sources = []
    clearTimeout(loopTimer)
    loopTimer = 0
    setState("stopped")
    setStartedAt(0)
  }

  function position() {
    if (state() !== "playing") return 0
    return audioContext().currentTime - startedAt()
  }

  return { state, position, play, stop }
}
```

### Task 3.4: Delete resets songLength when last clip removed

**Files:**
- Modify: `src/state.ts`

- [ ] **Step 1: Hook deleteSelection or clip removal**

In `src/state.ts`, the current `deleteSelection` removes a layout node. We need to also drop the clip(s) belonging to that node and reset `songLength` if no clips remain.

Find `deleteSelection` (around line 221). Before removing the layout node, collect the entity ids in the subtree being removed. After the layout mutation, drop their clips:

```typescript
function deleteSelection() {
  const selection = app.selection
  if (selection === null) return
  const targetedPath = selection.path.slice(0, selection.path.length - selection.depth)

  // Collect entity ids in the subtree being removed.
  const removedIds: string[] = []
  function collectIds(node: Node) {
    if (node.type === "entity") {
      removedIds.push(node.id)
    } else {
      node.children.forEach(collectIds)
    }
  }
  const targetNode = resolveNode(app.layout, targetedPath)
  if (targetNode) collectIds(targetNode)

  const parentPath = targetedPath.slice(0, -1)
  const nextSelection: Selection = { path: parentPath, depth: 0 }

  setApp(app => {
    const next = removeAt(app.layout, targetedPath)
    app.layout = next ?? createEntity()
    app.selection = nextSelection
  })

  for (const id of removedIds) {
    clips.clearClip(id)
  }
  if (Object.keys(clips.clips()).length === 0) {
    setSongLength(null)
  }
}
```

## Notes for the implementer

- **Pruning ported files:** the eddy-1.0 files were written for a worker-per-track architecture with ring buffers. Anywhere you see references to `Worker`, `MessagePort`, `SharedArrayBuffer`, `AudioWorkletNode`, or `RingBuffer`, the MVP doesn't need it — delete the branch. If pruning becomes painful (the over-engineering is woven through), write the module fresh from the WebCodecs / mediabunny / Web Audio docs rather than fighting the port. The goal is *simple*, not *line-for-line faithful*.

- **Browser support:** WebCodecs and `VideoFrame` interop with WebGL `texImage2D` work in Chrome ≥ 94 on Android. Don't worry about other browsers in v1.

- **Mobile vs desktop in tests:** tests run in headless Chromium at 1280×800 (desktop viewport). Mobile-specific issues (touch handling, viewport units, performance) are validated by hand in Step 2 of each checkpoint.

- **When in doubt about API shape:** the spec at `docs/superpowers/specs/2026-05-11-eddy-mvp-design.md` is authoritative. Reach for the spec before improvising.
